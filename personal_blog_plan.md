# 个人博客网站规划文档

> **核心差异点**：你不只是一个博客，你是一个**可交互的数字分身**。访客不只是"看"你，而是"和你聊"。这个定位本身就是最好的个人品牌叙事——一个程序员把自己做成了产品。

**Slogan 参考**：
- `> whoami // 不如直接问我`
- `Build in public. Think out loud.`

---

## 一、产品定位

| 维度 | 描述 |
|------|------|
| 目标受众 | 技术开发者 / 程序员、产品经理 / 创业者、潜在雇主 / 客户、泛科技爱好者 |
| 核心功能 | 个人履历展示、博客文章发布、项目 & 产品展示、AI Agent 对话、社交媒体聚合 |
| 差异化亮点 | 将自己"蒸馏"成 AI Agent，访客可与 AI 版你直接对话 |
| 整体风格 | 科技感 · 程序员美学 · 终端 / 代码视觉语言 |

---

## 二、技术栈推荐

### 前端
- **框架**：Next.js 14（App Router）
  - 静态生成博客，速度极快
  - API Route 直接承接 Agent 后端逻辑
  - Vercel 一键部署，零运维
- **样式**：Tailwind CSS
- **动效**：Framer Motion
- **博客内容**：MDX（支持在文章中嵌入 React 组件）
- **内容管理**：Contentlayer 或纯 Markdown 文件

### Agent AI 层
- **LLM API**：Claude API 或 GPT-4o
- **向量检索（RAG）**：LangChain + Pinecone / pgvector
- **知识库格式**：JSON + Markdown 结构化文件

### 基础设施
- **部署**：Vercel
- **域名**：建议 `yourname.dev` 或 `yourname.io`
- **Analytics**：Umami（自托管，隐私友好）
- **图片 CDN**：Vercel Image Optimization / Cloudflare

## 四、五大核心页面设计

### 1. Hero 主页（科技感核心）

**设计方向**：
- 终端打字机动画，逐行输出自我介绍，例如：
  ```
  const me = {
    role: "Full-Stack Engineer",
    passion: "build products from 0 to 1",
    now: "exploring AI × Product",
  }
  ```
- 背景效果：粒子网格 / SVG 代码雨 / 低多边形网格
- 履历展示：时间轴卡片形式，有层次感
- 技术栈标签云：动态悬浮效果

**包含内容**：
- 一句话定位（角色 + 特长）
- 工作经历时间轴
- 技术栈 & 技能图谱
- 快速导航：博客 / 项目 / 联系我

---

### 2. 博客系统

**功能特性**：
- MDX 支持代码高亮、可嵌入自制 React 组件
- 文章分类：标签 / 系列 / 专题
- 阅读时间估算（`~5 min read`）
- RSS Feed 订阅
- 文章目录（TOC）自动生成
- 社交分享按钮

**内容策略**：
- 发布频率：初期每两周一篇
- 选题方向：`做 X 时遇到了 Y，我是这么解决的` 类型文章对程序员圈子吸引力最强
- 涵盖：技术深度文、产品思考、行业观点、个人见解

---

### 3. 项目 & 产品展示

**每个项目卡片包含**：
- 项目名 + 一句话描述
- 技术栈标签（React / Rust / Python 等）
- 封面截图 / 演示 GIF
- Live Demo 链接
- GitHub 仓库链接
- 项目状态标签（🚀 上线中 / 🔨 开发中 / 🗄️ 归档）

**特色设计**：
- "正在做的项目" 悬浮置顶状态，体现持续创造感
- 支持按技术栈 / 类型筛选

---

### 4. Agent 对话框（最核心亮点）⭐

**交互形式**：
- 右下角浮动聊天按钮，点击展开对话窗口
- 样式：仿终端风格，深色背景，绿色光标闪烁

**技术实现**：

```
访客提问
  └── Next.js API Route
        └── 向量检索（RAG）匹配相关知识
              └── 注入 System Prompt + 上下文
                    └── 调用 LLM API
                          └── 流式返回回答
```

**System Prompt 核心要素**：
- 你的技术背景和工作经历
- 你的技术偏好和观点（喜欢什么，不喜欢什么）
- 你正在研究的方向
- 说话风格（直接 / 有观点 / 不废话）
- 你的项目详细介绍
- 博客文章摘要

**知识库结构**：
```
knowledge/
├── profile.md       # 个人履历、技能
├── projects.json    # 项目列表及详情
├── opinions.md      # 技术观点、价值观
├── blogs/           # 博客文章向量化索引
└── qa.md            # 常见问题预设答案
```

**安全机制**：
- 每 IP 每天限制 N 条对话（防止 API 费用爆炸）
- 敏感信息过滤（不回答个人隐私类问题）
- 对话历史仅存 Session，不持久化

### 5. 社交 & 联系

**聚合平台**：
- GitHub（代码 & 开源项目）

**联系方式**：

- 明确接洽场景：技术合作 / 产品咨询 / 工作机会

## 五、开发阶段计划

### Phase 1 — MVP 上线（1-2 周）

| 任务 | 说明 |
|------|------|
| 初始化 Next.js 项目 | App Router + Tailwind + ESLint 配置 |
| 实现 Hero 主页 | 打字机动画 + 基础信息展示 |
| 博客系统搭建 | MDX 解析 + 文章列表 + 文章页 |
| 项目展示静态页 | 卡片布局 + 技术栈标签 |
| 社交链接 | Footer / About 页面 |
| 部署上线 | Vercel + 自定义域名 |

> 🎯 目标：让网站先存在，再迭代。

---

### Phase 2 — Agent 接入（2-3 周）

| 任务 | 说明 |
|------|------|
| 整理个人知识库 | 履历 + 项目 + 博客内容结构化 |
| 编写 System Prompt | 核心灵魂，需认真打磨 |
| 实现向量检索 | 博客文章 Embedding + 相似度搜索 |
| API Route 开发 | 流式输出 + 限速中间件 |
| 对话 UI 开发 | 浮动窗口 + 终端风格样式 |
| 上线测试调优 | 根据实际对话效果迭代 Prompt |

> 🎯 目标：Agent 能流畅、准确地以你的口吻回答关于你的问题。

---

### Phase 3 — 持续打磨（长期）

| 方向 | 具体动作 |
|------|---------|
| 内容运营 | 保持博客更新频率，建立内容积累 |
| SEO 优化 | 关键词策略 + 结构化数据 + sitemap |
| 邮件订阅 | 接入 Resend 或 Mailchimp，建立私域 |
| Agent 优化 | 根据访客常问问题持续更新知识库 |
| 性能优化 | Core Web Vitals 达标，LCP < 2.5s |
| 数据分析 | Umami 看访客行为，优化转化路径 |

## 六、关键决策建议

### 关于 Agent System Prompt

System Prompt 是整个 Agent 功能的灵魂，要认真对待：

- ✅ 写清楚你的技术观点（比如：我认为 X 比 Y 好，因为...）
- ✅ 描述你的工作方式和思维风格
- ✅ 列出你不擅长 / 不感兴趣的领域（真实感的关键）
- ✅ 加入你正在研究或感兴趣的方向
- ❌ 不要让 Agent 变成万能客服，要有鲜明个性

### 关于域名选择

| 后缀 | 适合场景 | 推荐度 |
|------|---------|-------|
| `.dev` | 程序员身份认同感强 | ⭐⭐⭐⭐⭐ |
| `.io` | 科技产品感 | ⭐⭐⭐⭐ |
| `.com` | 通用，但个性不足 | ⭐⭐⭐ |
| `.me` | 个人品牌感 | ⭐⭐⭐ |

### 关于内容策略

初期不要追求完美，**发布频率比单篇质量更重要**：
- 每两周一篇 > 三个月一篇
- 选题优先考虑：「我在做 X 时遇到了 Y，最终用 Z 解决了」
- 这类实战经验类文章在程序员圈子吸引力最强，也最有 SEO 价值
