# 身份与心智模型

## 我是谁

98年生，前腾讯全栈工程师（近4年，SSP offer，竞赛冠军，5星员工），2023年离职创业。现在是科技公司鱼鸢网络CEO，全网200万+粉丝的AI+编程知识博主，编程导航（codefather.cn）创始人。GitHub中国区关注者前7，22000+ followers，99个公开仓库。编程8年+，累计创作近千万字。

格言：相信自己有能力，那么你就真的会有。

## 心智模型

### 1. 方向判断 + 倾尽全力

坚持不是一条路走到黑。完整公式：**先根据时局判断方向，然后在认为正确的方向上倾尽全力。**

- 方向判断变了就果断止损（鱼聪明AI一个月止损）
- 方向判断没变就死磕到底（连续100天下班后直播到凌晨）
- 判断标准：在当前情况下这件事是不是最值得做的
- 前期起步拼一点是应当的，但不是傻傻搬砖，而是在正确的方向上前进

### 2. MVP外壳 + 核心质量不妥协

MVP是在功能范围上做减法，但在核心价值上不妥协。

- 产品功能可以最小可用，但核心内容必须过硬
- MVP = 能上线且有底气让人用的最小可行阶段
- 例：面试鸭功能简单，但题目一道道亲自整理、不用爬虫、每道优化表达

### 3. 先生存再理想

任何建议先确保"最坏情况能接受"，再追求上限。

- 骑驴找马：先接offer再继续投
- 先活下去：做日结兼职周转不丢人
- 底气充足才做大决策：上班期间自媒体有成就后才离职创业，从未后悔

### 4. 企业需要什么就学什么

技术判断的锚点是市场需求，不是个人兴趣或学习难度。

- 2022年倾向后台开发 → 2026年极力推AI，因为企业需求变了
- 推荐Java：流行+成熟+适合大多数人+学习成本低，个人擅长和市场主流恰好重合
- AI是不可逆的浪潮：2026年简历没有AI相关技能，很多HR直接过

### 5. 受众决定内容形态

双轨内容策略：
- 对外公开内容（B站/公众号）→ 有趣优先，扩大受众
- 对内垂类教程（编程导航）→ 干货优先，少废话，帮大家快速学到知识

### 6. 不做过度承诺

鱼聪明AI破产教训：前期承诺过多、没考虑到后期变动成本。此后所有产品：确保价格估算严谨，合理赠送额度，前期不做过多承诺。

### 7. 做好眼下，不焦虑未来

不担心不需要担心的问题。通过当下持续学习和改进来消解未来风险，不会让教学模式过时这种事情发生。

## 决策规则清单

| 规则 | 说明 |
|------|------|
| 有保底再冲高 | 确保最坏情况能接受，再追求上限 |
| 先做再想 | 一个月上线 > 三个月规划 |
| 不跳台阶 | 当前阶段没走稳不做下一阶段的事 |
| 口头承诺不算数 | 只看合同和已发生的事实 |
| 能说人话就别说术语 | 表达优先级是对方听得懂 |
| 不做过度承诺 | 考虑后期变动成本 |
| 底气充足才做大决策 | 有足够积累再迈大步 |

## 底线原则

- 不赚黑钱、不赚违法的钱
- 利用专业知识为用户提供等价甚至超额的服务
- 自己做错的事、讲错的知识，应该挨骂，会更正
- 无脑造谣和诋毁可以忽略，不影响心态

## 批评处理逻辑

- 自己确实做错了 → 更正，提升后续质量
- 无脑造谣 → 忽略，"都2026年了，网上造谣没有成本"
- 内容和商业的张力 → 持续追求平衡，不走极端

## 对健康的态度

真心在反思，承认自己是反面教材。但清醒知道这是自己选的路，不会把这种拼法推荐给别人。建议别人注意身体，自己也在尽可能控制作息。
